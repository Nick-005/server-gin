// @Summary         - Краткое описание

// @Description     - подробное описание

// @Tags users      - сортировка по тэгам  

// @Accept  json    - что ожидает на вход (только формат (json,xml,form,multipart,plain))

// @Produce  json   - что выдает в ответе (только формат (json,xml,form,multipart,plain))

// @Param <название> <тип параметра> <тип данных> <обязательность> <описание>
<название>: Имя параметра.
<тип параметра>: Тип параметра (например, query, path, body, header).
<тип данных>: Тип данных параметра (например, string, int, object, array).
<обязательность>: Указывает, является ли параметр обязательным (true или false).
<описание>: Краткое описание параметра.

// @Success <HTTP-код> {<тип данных>} <описание>
<HTTP-код>: HTTP-статус код успешного ответа (например, 200, 201).
{<тип данных>}: Тип данных, возвращаемых в ответе (например, object, array, string).
<описание>: Краткое описание ответа.

// @Failure <HTTP-код> {<тип данных>} <описание>
<HTTP-код>: HTTP-статус код ошибки (например, 400, 404, 500).
{<тип данных>}: Тип данных, возвращаемых в ответе (например, object, string).
<описание>: Краткое описание ошибки.

// @Failure 500 {object} map[string]string

// @Router /users [post]    - Используется для указания HTTP-метода и пути эндпоинта.
@Router <путь> [<HTTP-метод>]
<путь>: Путь эндпоинта (например, /users, /users/{id}).
[<HTTP-метод>]: HTTP-метод (например, get, post, put, delete).
